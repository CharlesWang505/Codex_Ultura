# 更新日志

本项目采用语义化版本。每次代码、界面或构建配置修改都必须新增一条版本记录，并同步递增应用版本号。

## [1.2.10] - 2026-07-12

### 热切换模型重新注入

- 修复自动路由开启时模型目录仍只读取当前单个供应商，导致多供应商映射中 GPT 可以显示、Claude 等其他供应商模型未进入 Codex 白名单的问题。
- 热切换模型目录现在一次返回全部有效映射，并以当前映射集合校正默认模型。
- 重新生成模型后会主动从 Codex 的模型响应、React 状态、Statsig 配置和可用模型集合中移除未选供应商的旧模型，避免 Grok、Gemini 等上一次注入内容残留但无法使用。
- 新增多供应商模型目录和旧供应商模型排除的回归测试。

## [1.2.9] - 2026-07-12

### Claude 流式响应

- 修复部分 Anthropic 原生中转在收到 `stream=true` 时仍返回完整 JSON，网关只生成 `response.created` 和 `response.completed`，导致 Codex 模型请求成功但对话界面不显示回答的问题。
- 完整 JSON 转 Responses SSE 时现在补齐 `response.in_progress`、输出项、内容块、文本增量和完成事件。
- 同步兼容非流式回退下的文本、推理内容、函数调用和自定义工具调用。
- 新增完整响应转换为 Codex 文本事件序列的回归测试。

## [1.2.8] - 2026-07-12

### 供应商仅保存

- 修复供应商点击“仅保存”时，旧 `config.toml` 中的模型优先于界面当前默认模型，导致默认模型被强制改回旧值的问题。
- 保存归一化现在优先采用界面明确选择的默认模型；仅在默认模型为空时，才回退读取已有配置或模型列表首项。
- 修复旧模型被重新插入已获取模型列表首位，造成模型数量无故增加的问题。
- 新增默认模型与旧配置不一致时的保存回归测试。

## [1.2.7] - 2026-07-12

### Codex 模型选择

- 对齐 Codex SwitchGate 的模型目录机制：本地网关识别带 `client_version` 的模型请求，并返回包含显示名、上下文窗口和推理能力的 Codex catalog。
- 普通供应商的模型列表即使没有 `[1M]`、`[200K]` 等上下文后缀，也会生成 `model_catalog_json` 并写入 `config.toml`，不再只显示单个默认模型。
- 模型选择器显示“模型 · 供应商”，便于区分相同模型在不同供应商下的路由。
- 模型目录提供极低、轻度、中等、高、极高和最高六档推理强度，协议代理继续按供应商方言转换对应参数。

### 日志准确性与性能

- 修复 New API `other.frt` 返回毫秒值时被误当作秒，导致首字耗时被裁剪成总响应时间的问题。
- 日志分页并发由 6 提升到 12；由于站点强制每页最多返回 100 条，每批最多并行获取约 1200 条。
- 进度更新改为按并发批次合并，减少大批量日志加载时的重复解析和界面刷新。
- 兼容部分 New API 站点 `p=0` 与 `p=1` 都返回第一页的行为，避免重复第一页并遗漏最后一页。

### 验证

- Codex catalog、普通模型列表配置和供应商切换相关 Rust 测试通过：109 项通过、0 项失败。
- 前端 Codex UI 测试、Lint、TypeScript 检查、生产构建及 Windows x64 桌面打包通过。
- 全量 Rust 核心测试中，本次相关测试均通过；仍有 1 项既有的内置插件市场快照测试失败，与本次修改无关。

## [1.2.5] - 2026-07-11

### 日志响应时间

- 修复 New API / One API 将 `first_response_time` 返回为绝对时间戳时，被误当作首字耗时并最终钳制成总响应时间的问题。
- 首字耗时现在优先使用“首包时间戳 - 请求创建时间”，同时兼容直接返回秒数、毫秒数以及 TTFT/FRT 字段的旧接口。
- 日志表将“用时/首字”改为更清晰的“响应耗时”，两个徽标增加“总”和“首”标识，并适当加宽时间列。
- 首字状态颜色调整为：5 秒以内良好、5-10 秒提醒、超过 10 秒偏慢。

### 多供应商模型注入

- 修复热切换只生成模型目录、未确保 Codex 增强脚本重新注入的问题。
- 选定供应商生成模型时自动启用“模型白名单解锁”，并通过内置启动器请求重启 Codex。
- 热切换开启或更新时始终重写 Codex 网关配置，避免模型目录指针被其他配置覆盖后无法恢复。
- 状态由“已注入”改为准确的“已生成”，不再把映射条数当作 Codex 已加载数量。

## [1.2.4] - 2026-07-11

### 原生协议 Base URL

- 修复 Anthropic、Gemini 等原生协议供应商保存后，内部协议转换地址 `http://127.0.0.1:58321/v1` 被错误回显为上游 Base URL 的问题。

- 修复默认模型点击“仅保存”后虽然已写入配置，但重新加载时被 IPC 序列化隐藏、界面显示为空的问题。
- 8787 本地网关配置区新增独立“保存配置”按钮，只保存固定供应商、模型、推理强度和自动路由开关，不启动网关、不修改 Codex。

### 使用提示

- New API、One API 等 OpenAI 兼容中转站即使 Key 属于 Claude 分组，也应优先选择 OpenAI Responses；只有上游明确提供 Anthropic `/v1/messages` 原生接口时才选择 Anthropic 原生。
